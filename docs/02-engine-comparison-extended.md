# 补充引擎选项

---

## 6. Defold

| 维度 | 说明 |
|------|------|
| 语言 | Lua |
| 授权 | 完全免费，Apache 2.0 |
| 2D 支持 | 原生 2D，轻量高效 |
| 上手难度 | 低-中 |
| 社区 | 中等，King（糖果传奇开发商）支持 |
| 特点 | 极小包体（<5MB），跨平台优秀，HTML5 导出性能好 |
| 格斗游戏案例 | 较少，需自建所有系统 |
| 回滚网络 | 需自集成 GGPO 或自建 |
| 适合场景 | 移动端/Web 轻量级格斗游戏 |

**优势：**
- 包体极小，适合 Web/移动端发布
- Lua 脚本简单易学
- 性能优秀，内存占用低

**劣势：**
- 格斗游戏生态几乎为零，所有系统需从零搭建
- 没有现成的回滚网络插件
- 可视化编辑器功能较基础

---

## 7. GameMaker Studio 2

| 维度 | 说明 |
|------|------|
| 语言 | GML（GameMaker Language） |
| 授权 | 订阅制（$5-10/月）或买断（$100+） |
| 2D 支持 | 原生 2D，历史悠久 |
| 上手难度 | 低（拖拽式 + 脚本） |
| 社区 | 大，独立游戏开发者众多 |
| 格斗游戏案例 | Rivals of Aether（知名独立格斗游戏） |
| 回滚网络 | 社区有第三方实现，但不成熟 |
| 适合场景 | 2D 独立游戏快速原型 |

**优势：**
- 拖拽式开发，上手极快
- Rivals of Aether 证明可行性
- 大量 2D 游戏教程

**劣势：**
- GML 语言生态封闭，无法复用外部库
- 回滚网络需自建，难度高
- 订阅费用长期累积成本高

---

## 8. Cocos2d-x

| 维度 | 说明 |
|------|------|
| 语言 | C++ / Lua / JavaScript |
| 授权 | MIT，完全免费 |
| 2D 支持 | 原生 2D |
| 上手难度 | 中-高（C++ 复杂度） |
| 社区 | 中国区活跃，国际社区衰退 |
| 格斗游戏案例 | 早期手游格斗游戏有使用 |
| 回滚网络 | 需自集成 GGPO |
| 适合场景 | 移动端 2D 游戏（尤其中国市场） |

**优势：**
- 中国区文档和社区支持好
- 移动端性能优秀
- C++ 底层可深度优化

**劣势：**
- 国际社区活跃度下降
- 相比 Godot/Unity 现代化程度不足
- 格斗游戏生态几乎为零

---

## 9. MonoGame / FNA

| 维度 | 说明 |
|------|------|
| 语言 | C# |
| 授权 | MIT，完全免费 |
| 2D 支持 | 框架级（非引擎），需自建渲染管线 |
| 上手难度 | 高（接近自研引擎） |
| 社区 | 中等，XNA 老用户群体 |
| 格斗游戏案例 | Skullgirls（使用 Z-Engine，基于 XNA） |
| 回滚网络 | 可集成 Backdash（C# 回滚库） |
| 适合场景 | 有 C# 经验，想完全掌控底层的团队 |

**优势：**
- Skullgirls 证明可行性（商业级格斗游戏）
- C# 生态成熟，可复用 .NET 库
- 完全掌控渲染和逻辑

**劣势：**
- 不是引擎，是框架，需自建大量系统
- 学习曲线陡峭
- 缺少可视化编辑器

---

## 10. Phaser（Web 专用）

| 维度 | 说明 |
|------|------|
| 语言 | JavaScript / TypeScript |
| 授权 | MIT，完全免费 |
| 2D 支持 | 原生 2D，Canvas/WebGL |
| 上手难度 | 低 |
| 社区 | 大，Web 游戏主流框架 |
| 格斗游戏案例 | 少，主要用于休闲游戏 |
| 回滚网络 | 可集成 GGPO.js 或自建 |
| 适合场景 | 纯浏览器格斗游戏 |

**优势：**
- Web 原生，无需安装
- TypeScript 类型安全
- 快速迭代

**劣势：**
- 性能受浏览器限制
- 不适合桌面/主机发布
- 格斗游戏生态几乎为零

---

## 11. Stride（原 Xenko）

| 维度 | 说明 |
|------|------|
| 语言 | C# |
| 授权 | MIT，完全免费 |
| 2D 支持 | 基于 3D 引擎的 2D |
| 上手难度 | 中-高 |
| 社区 | 小，但在增长 |
| 格斗游戏案例 | 无 |
| 回滚网络 | 可集成 Backdash |
| 适合场景 | 3D 为主，2D 为辅的项目 |

**优势：**
- 开源，社区驱动
- C# 生态
- 现代渲染管线

**劣势：**
- 2D 不是重点
- 社区小，资源少
- 格斗游戏无先例

## 最终建议更新

**如果你想要：**
- 最快上手 + 免费 → **Godot 4**
- 商业级生态 + 不在意费用 → **Unity 6**
- 技术最先进 + 有 Rust 经验 → **Bevy**
- Web 原生 + 轻量级 → **Defold** 或 **Phaser**
- 完全掌控底层 + C# 经验 → **MonoGame/FNA**
- 已有成功案例参考 → **GameMaker**（Rivals of Aether）或 **MonoGame**（Skullgirls）

**不推荐：** Cocos2d-x（生态衰退）、Stride（2D 非重点）
