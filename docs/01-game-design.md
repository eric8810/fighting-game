# 游戏策划设计调研

> 参考：拳皇 2000（King of Fighters 2000）及同类 2D 格斗游戏

---

## 1. 核心战斗机制

### 1.1 帧系统

所有 2D 格斗游戏以固定 60FPS 运行，每个动作拆分为三段：

| 阶段 | 说明 |
|------|------|
| 启动帧（Startup） | 动作开始到攻击判定出现前的帧数 |
| 攻击帧（Active） | 攻击判定存在的帧数 |
| 恢复帧（Recovery） | 攻击结束到角色可再次行动的帧数 |

帧优势（Frame Advantage）= 对手硬直帧 - 己方恢复帧，正值有利，负值不利。

### 1.2 碰撞盒系统

| 类型 | 说明 |
|------|------|
| Hitbox（攻击框） | 攻击判定区域，与对手 Hurtbox 重叠则命中 |
| Hurtbox（受伤框） | 角色可被攻击的区域 |
| Pushbox（推挤框） | 防止角色重叠，控制间距 |
| Throwbox（投技框） | 投技专用判定框 |
| Proximity Guard（近距防御框） | 靠近时自动进入防御姿态的触发框 |

### 1.3 输入系统

- **输入缓冲**：约 8-16 帧宽容窗口，允许提前输入
- **指令类型**：
  - QCF（↓↘→）：波动拳类
  - QCB（↓↙←）：反向波动
  - DP（→↓↘）：升龙拳类
  - HCF/HCB（←↙↓↘→）：半圆前/后
  - 360°（一圈）：投技类超必杀
  - 双击方向（→→）：跑步/冲刺

## 2. KOF 系列特色机制

### 2.1 跑步系统（Run System）
- →→ 触发跑步，可取消为攻击
- 跑步中可接普攻/特殊技，形成跑步连段

### 2.2 紧急回避（Emergency Evasion / Roll）
- ←← 或 →← 触发翻滚，带无敌帧（约 10-15 帧）
- 可用于穿越攻击、接近对手

### 2.3 气槽系统（Power Gauge）
- 共 3 股（MAX 3000 点），受击/攻击均可充能
- 消耗 1 股：发动超必杀（DM, Dream Move）
- 消耗 2 股：发动 SDM（Super Dream Move），伤害更高
- 消耗 1 股：激活 MAX 模式

### 2.4 MAX 模式（MAX Mode）
- KOF 2001 引入，KOF 2000 为 HSDM 前身
- 激活后限时强化，技能伤害提升，可发动 MAX 超必杀
- 气槽持续消耗，耗尽或时间到则退出

### 2.5 Striker 系统（KOF 2000 独有）
- 队伍中后备角色可作为 Striker 召唤助攻
- 消耗 1 股气，Striker 出场执行固定动作后退场
- 每场比赛可使用次数有限

### 2.6 连招取消系统（Cancel Chain）
- 普攻 → 特殊技（Special Cancel）
- 特殊技 → 超必杀（Super Cancel）
- 超必杀 → MAX 超必杀（MAX Cancel，KOF 2002 UM 特色）
- 取消窗口由帧数据定义

## 3. 角色设计考量

### 3.1 角色原型分类

| 类型 | 特点 | 代表角色 |
|------|------|----------|
| 均衡型 | 各项属性均衡，适合新手 | 草薙京、李白龙 |
| 强攻型 | 高伤害，低防御，连段复杂 | 克里斯、山崎龙二 |
| 防守反击型 | 高防御，反击技能强 | 大门五郎 |
| 投技型 | 近距离投技为主 | 拉尔夫、克拉克 |
| 远程型 | 弹幕/远距离技能强 | 不知火舞、比利 |
| 机动型 | 移动速度快，连段灵活 | 二阶堂红丸 |

### 3.2 平衡性原则
- 每个角色至少有 1 个强力工具（Poke/连段/防守）
- 强力技能需有明显弱点（启动慢/恢复长/范围小）
- 避免单一最优解（Dominant Strategy）
- 通过帧数据微调而非重设计来平衡

## 4. 场景设计

| 要素 | 说明 |
|------|------|
| 舞台宽度 | 约 3-4 屏宽，角色不可出界 |
| 视差背景 | 多层滚动，增加纵深感 |
| 边界墙 | 角色被逼到角落时攻防转换 |
| 动态元素 | 背景 NPC、动态光影（可选，不影响游戏性） |
| 音乐 | 每个舞台独立 BGM，节奏与战斗感匹配 |
