# 引擎与技术栈选型调研

---

## 1. 引擎横向对比

| 维度 | Godot 4 | Unity 6 | Bevy (Rust) | 自研引擎 |
|------|---------|---------|-------------|----------|
| 语言 | GDScript / C# / C++ | C# | Rust | 自选 |
| 授权费用 | MIT，完全免费 | 个人免费，商业抽成 | MIT/Apache，免费 | 无授权费 |
| 2D 支持 | 原生 2D 引擎 | 基于 3D 改造的 2D | 原生 2D 渲染器 | 完全自定义 |
| 上手难度 | 低-中 | 中 | 高（Rust 所有权） | 极高 |
| 社区生态 | 大，快速增长 | 极大，资源丰富 | 中，快速增长 | 无 |
| 格斗游戏框架 | Sakuga Engine、godot-mugen | UFE2（付费插件） | bevy_ggrs + GGRS | 完全自建 |
| 回滚网络 | rollback-netcode addon | Mirror + Rollback | bevy_ggrs（原生最佳） | 自集成 GGPO |
| 帧确定性 | 需手动保证（避免浮点） | 需手动保证 | ECS 天然利于确定性 | 自控 |
| 动画状态机 | AnimationTree（可视化 HFSM） | Animator / Animancer | 自建系统 | 自建 |
| WebAssembly | 支持 | 支持 | 支持（wasm32） | 自建 |
| 开发迭代速度 | 快（脚本热重载） | 中（编译较慢） | 慢（Rust 编译） | 极慢 |
| 适合独立开发 | ★★★★★ | ★★★★ | ★★★ | ★ |

## 2. 各方案详细评析

### 2.1 Godot 4（推荐首选）

**优势：**
- 完全免费，MIT 授权，无商业抽成风险
- 原生 2D 引擎，像素单位，无 3D 开销
- AnimationTree 内置可视化 HFSM，无需额外插件
- Area2D 节点系统天然适合多类型碰撞盒管理
- GDScript 类 Python 语法，快速原型；性能瓶颈可切换 C#
- 社区活跃，格斗游戏相关插件持续增长

**劣势：**
- GDScript 性能不及 C++/Rust
- 大型项目 C# 编译时间略长
- 相比 Unity 商业资源较少

**适用场景：** 独立开发者、小团队、预算有限的项目

---

### 2.2 Unity 6

**优势：**
- 生态最成熟，资源商店丰富
- UFE2（Universal Fighting Engine 2）提供完整格斗框架
- 大量教程和社区支持

**劣势：**
- 2019-2024 年多次调整商业授权政策，长期稳定性存疑
- 2D 管线本质是 3D 引擎子集，有不必要的性能开销
- UFE2 约 $200-400，且非开源（黑盒）
- 编译速度较慢，迭代效率低于 Godot

**适用场景：** 已有 Unity 经验的团队，或需要大量商业资源的项目

---

### 2.3 Bevy（Rust）

**优势：**
- ECS 架构天然支持确定性逻辑，与回滚网络完美契合
- bevy_ggrs 是目前最先进的开源回滚网络实现之一
- Matchbox + WebRTC 支持浏览器 P2P 联机（WASM）
- Rust 内存安全，无空指针，无数据竞争（编译时保证）
- 并行系统自动调度，性能极高

**劣势：**
- Rust 学习曲线陡峭（所有权、借用检查器）
- ECS 设计模式需要转变思维方式
- 生态相较 Godot/Unity 仍在成长中
- 编译时间较长（增量编译已改善）
- 缺乏可视化编辑器（需配合 Blender 或自建工具）

**适用场景：** 有 Rust 经验的技术型团队，追求极致性能和网络质量

---

### 2.4 自研引擎

**优势：**
- 完全掌控所有细节
- 无第三方依赖风险

**劣势：**
- 开发周期极长，需要从零实现渲染、物理、音频、输入等所有子系统
- 维护成本极高
- 不适合任何规模的独立项目

**适用场景：** 不推荐，仅适合有专业引擎团队的大型商业项目

---

## 3. 开源格斗引擎/框架评估

### 3.1 MUGEN（经典，不推荐新项目）

| 项目 | 状态 | 语言 | 说明 |
|------|------|------|------|
| 原始 MUGEN | 停止维护 | 专有脚本（CNS） | Elecbyte 出品，已停更 |
| IKEMEN GO | 活跃维护 | Go + Lua | 兼容 MUGEN 1.1 资源，MIT 授权 |
| Ikemen Plus Ultra | 低活跃 | C++ + Lua | 社区分支，功能更多 |

**适用场景：** 想复用大量 MUGEN 社区资源（角色/场景素材）
**不适用：** 从零设计新系统、现代网络联机、现代化架构

---

### 3.2 Sakuga Engine（推荐的现代开源格斗框架）

- **基于：** Godot 4 + C#
- **特性：** 内置回滚网络、帧数据编辑器、多类型碰撞盒管理
- **定位：** 专为 KOF/街霸风格 2D 格斗设计
- **授权：** MIT
- **适合：** 基于 Godot 生态构建原创格斗游戏

---

### 3.3 OpenBOR

- **定位：** Final Fight / 双截龙类 Beat 'em Up 引擎（非 1v1 对打）
- **语言：** C + 专有脚本
- **注意：** 更适合多人横版动作游戏，不适合本项目

---

### 3.4 Backdash（C# 回滚 SDK）

- **定位：** 纯 C# 的 GGPO 风格回滚网络 SDK，不绑定特定引擎
- **兼容：** Unity / Godot / MonoGame
- **适合：** 已有 Unity 项目，想单独添加回滚网络能力

---

## 4. 最终推荐

| 场景 | 推荐方案 |
|------|----------|
| 独立开发者 / 小团队 | **Godot 4 + GDScript/C# + rollback-netcode** |
| 有 Rust 经验的技术团队 | **Bevy + GGRS + Matchbox** |
| 已有 Unity 项目 | **Unity 6 + Backdash 或 UFE2** |
| 想用 MUGEN 素材 | **IKEMEN GO** |
