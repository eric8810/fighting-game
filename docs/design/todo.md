## 阶段零：定义你的游戏是什么

> 在写任何帧数据或参数之前，先回答根本问题。
> 这些答案决定了后续所有设计——错了可以改，但不能没有。

### 0.1 体验支柱（Experience Pillars）

用 3 个关键词定义"玩家打完一局后应该是什么感觉"。
所有后续设计决策都要对照这三个词验证。

- [x] 写下 3 个手感关键词
- [x] 为每个关键词写一句话解释：它在对战中具体表现为什么

### 0.2 五个根本维度定位

在每个维度上明确你的游戏站在什么位置，画出你自己的"雷达图"。

- [x] **空间博弈**：远近切换——拉远牵制和突然贴身交替出现
- [x] **风险/回报结构**：大开大合——赌对赚大，赌错也痛
- [x] **势头与翻盘**：随时可翻——落后方有逆转机会，领先不安全
- [x] **表达空间**：双高——连段+战术+角色差异极端化
- [x] **张力曲线**：玩家主动点燃——选择时机赌一把

### 0.3 竞品研究（服务于定位，而非照抄）

带着 0.2 的问题去研究，而非记录数据本身。

- [x] 选 3 款参考游戏：KOF XIII（直接竞品）、3rd Strike、街霸6
- [x] 以 KOF XIII 为基准逐维度对比
- [x] 总结：KOF XIII 的手感 + Guilty Gear 级别的角色差异化

### 0.4 核心机制决策

基于 0.2 的定位，决定需要什么机制来实现它。不是照搬某个游戏的系统列表，而是问"我需要什么工具来实现我想要的体验"。

- [x] 空间博弈 → 通用移动（KOF XIII框架）+ 角色专属移动（必杀技=移动工具）
- [x] 风险/回报 → 全面大开大合（连段高伤+单发也重+防御选择有风险）
- [x] 势头与翻盘 → 攻防双通道逆转（进攻端爆发+防御端高风险选项，两条路翻盘）
- [x] 表达空间 → 角色设计阶段落实（GG级差异化）
- [x] 张力曲线 → 单一资源攻防博弈（进攻爆发和防御赌博争夺同一条气）

> **阶段零产出物**：`docs/design/pillars.md`——一页纸，包含体验支柱、五维定位、核心机制清单。
> 这是整个项目的设计宪法，后续所有争议回到这里裁决。

## 阶段一：最小可打原型

> 目标：用最少的内容验证阶段零的定位是否成立。方块就行，不需要美术。

### 1.1 移动与空间

不是"实现步行"，而是验证空间博弈的基础是否成立。

- [ ] 移动方式实现（步行、奔跑、后退、跳跃）
- [ ] **验证**：两个角色只能移动（不能攻击）时互相追逐，距离控制感觉是否符合 0.2 的空间定位？
- [ ] 调整移动参数直到满意（速度、跳跃弧线、奔跑刹车感）

### 1.2 基础攻防

不是"实现 4 个通常技"，而是验证风险/回报结构是否成立。

- [ ] 最小攻击集（一个快攻击、一个慢重攻击、一个下段、一个空中攻击）
- [ ] 防御（站防、蹲防）
- [ ] 投技（打破防守的手段）
- [ ] **验证**：攻防形成有意义的猜拳？进攻方和防守方都有选择？
- [ ] **验证**：打中人的感觉对不对？（顿帧、击退、音效的初步调校）

### 1.3 一个特殊招式

不是"实现 236 输入"，而是验证基础攻防之外是否需要更多层级。

- [ ] 一个需要特殊输入的招式（比通常技更强但更有风险）
- [ ] **验证**：这个招式创造了通常技无法覆盖的战术选择？它让对战更丰富了？

> **阶段一完成标准**：两个人打 10 分钟，攻防有来有回，且对战感觉符合阶段零定义的体验支柱。
> 如果不符合，调整参数或回到阶段零修改定位。不要带着错误的基础进入阶段二。

## 阶段二：核心系统

> 逐个加入机制，每加一个就回到体验支柱验证。
> 加入的顺序由阶段零 0.4 的机制清单决定，以下仅为示例。

### 2.1 连续技系统（如果你的表达空间需要它）

- [ ] 取消规则设计（什么能取消什么）
- [ ] 连段伤害补正
- [ ] **验证**：连续技的长度和伤害是否符合风险/回报定位？
- [ ] **验证**：连段是否提供了表达空间定位所需的创造力？

### 2.2 资源系统（如果你的张力曲线需要它）

- [ ] 资源获取规则
- [ ] 资源消耗选项（强化招式？超必杀？爆发模式？）
- [ ] **验证**：资源积累的节奏是否制造了张力曲线定位所需的"高潮时刻"？

### 2.3 防御选项扩展（如果你的风险/回报需要它）

- [ ] 高级防御选项（回避？弹开？格挡反击？）
- [ ] 防御惩罚机制（削气？Guard Crush？）
- [ ] **验证**：防守的安全程度是否符合风险/回报定位？

### 2.4 逆转机制（如果你的势头与翻盘需要它）

- [ ] 劣势方的逆转工具（爆气？无敌技？）
- [ ] 逆转工具的代价与限制
- [ ] **验证**：被压制方是否有阶段零定义的合理出路？不会太强也不会没用？

### 2.5 其他核心机制

- [ ] 阶段零 0.4 清单中尚未实现的机制
- [ ] 每个机制加入后都要回到体验支柱验证

> **阶段二完成标准**：对战体验完整地实现了阶段零的五维定位。
> 一个角色拥有所有核心系统，对战有深度和变化。

## 阶段三：角色设计

> 系统确定后，才能开始真正的角色设计。
> 角色是系统的"演奏者"——系统是乐器，角色是曲子。

### 3.1 角色设计框架

- [ ] 角色原型分类——不是照搬"近距离/远距离"，而是基于你的五维定位思考：
  - 在你的空间博弈中，不同角色可以在什么距离上有优势？
  - 在你的风险/回报结构中，不同角色可以偏向什么风格？
  - 这些差异如何产生有趣的对局而非单方碾压？
- [ ] 角色属性规范（标准值和允许偏差范围）
- [ ] 角色设计模板

### 3.2 基准角色

- [ ] 一个体现系统"标准玩法"的角色（平衡型，所有工具都有但不极端）
- [ ] 完整招式表、帧数据、判定框
- [ ] 连续技路线
- [ ] **验证**：用这个角色打，能体验到所有核心系统的乐趣？

### 3.3 对比角色

- [ ] 一个和基准角色玩法尽可能不同的角色
- [ ] 完整设计（同上）
- [ ] **验证**：两个角色对打，双方都觉得好玩？角色差异产生了新的战术层次？

> **阶段三完成标准**：2 个角色的对战涌现出阶段零未预见的有趣场景——这说明系统有生命力。

## 贯穿始终的原则

1. **所有设计决策回到体验支柱验证** — 纠结时问"哪个选项更符合我的三个关键词？"
2. **每次只改一个变量** — 否则你不知道什么改变了手感
3. **打比写重要** — 1 小时对战测试 > 3 小时写文档
4. **记录每次测试心得** — `docs/design/playtest-notes/` 下写日期标记的简短笔记
5. **允许回头** — 如果后续阶段的测试发现阶段零的定位有问题，改定位，不要硬撑
6. **KOF2000 是参考，不是目标** — 借鉴它验证过的东西，但你做的是你自己的游戏
