# 游戏黑盒测试 - 快速验证指南

## 🎯 测试目标
验证glyphon文本渲染集成是否成功，游戏是否可以正常运行并显示文本。

## ⚡ 快速测试步骤（5分钟）

### 步骤1：编译游戏
```bash
cd D:\code\tickle
cargo build --release
```
**预期**：编译成功，无错误

### 步骤2：启动游戏
```bash
cargo run --release
```
或直接运行：
```bash
.\target\release\game.exe
```

### 步骤3：验证主菜单
**观察窗口**：
- ✅ 窗口标题显示 "Tickle Fighting Engine"
- ✅ 看到大标题文字 "TICKLE FIGHTING ENGINE"（不是彩色方块）
- ✅ 看到三个菜单选项的文字：
  - VS MODE
  - TRAINING
  - QUIT
- ✅ 选中的选项是白色，未选中的是灰色

**如果看到彩色方块而不是文字** → ❌ 文本渲染失败

### 步骤4：进入游戏
按 **Space** 或 **Enter** 选择 "VS MODE"

**观察游戏画面**：
- ✅ 左上角看到绿色 "P1" 文字
- ✅ 右上角看到红色 "P2" 文字
- ✅ 顶部中央看到黄色数字 "99"（计时器）
- ✅ 计时器每秒递减：99 → 98 → 97...

### 步骤5：测试连击计数器
- 用 P1（WASD + Space）靠近 P2
- 连续按 Space 攻击 2-3 次
- ✅ 看到金色文字 "2 HITS" 或 "3 HITS"

### 步骤6：测试暂停菜单
按 **Escape** 暂停游戏

**观察暂停菜单**：
- ✅ 看到 "RESUME" 文字
- ✅ 看到 "QUIT TO MENU" 文字
- ✅ 选中项是白色

### 步骤7：测试回合结束
- 恢复游戏（按Space选择RESUME）
- 让P1击败P2（耗尽P2血量）
- ✅ 看到大横幅文字 "PLAYER 1 WINS!"

## 🐛 已知问题排查

### 问题1：游戏启动后立即崩溃（exit code 0xffffffff）

**可能原因**：
1. glyphon初始化失败
2. wgpu设备创建失败
3. 字体系统加载失败

**诊断方法**：
```bash
# 启用详细日志
set RUST_LOG=info
cargo run --release
```

查看日志中最后一条消息，确定崩溃点。

### 问题2：窗口打开但显示黑屏

**可能原因**：
- 渲染循环未启动
- 文本渲染器prepare()失败

**解决方法**：
检查日志中是否有 "resumed() called" 和 "All renderers initialized successfully"

### 问题3：看到彩色方块而不是文字

**可能原因**：
- `render_text()` 方法未被调用
- TextArea数据未正确传递给glyphon
- 字体加载失败

**解决方法**：
这是严重的集成问题，需要检查代码。

